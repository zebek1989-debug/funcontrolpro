# Closed Beta Recruitment Plan

Last updated: 2026-03-15

## Target cohort

- Minimum active testers: `10`
- Planned cohort size: `12` (buffer for dropout)
- Hardware mix target:
  - `>= 5` users with expected `Full Control`
  - `>= 5` users with expected `Monitoring Only`
  - up to `2` wildcard configurations

## Recruitment channels

1. Existing users from alpha/internal build.
2. Hardware enthusiast communities with clear disclosure.
3. Direct outreach to users with boards from supported-hardware list.

## Entry criteria for each tester

- Uses Windows machine supported by current release artifact.
- Agrees to send support bundle with each beta bug report.
- Confirms awareness that beta build can contain regressions.
- Provides hardware summary before onboarding.

## Onboarding checklist

1. Add tester to `docs/beta/tester-roster.csv`.
2. Send install package + release notes draft.
3. Share issue template and support bundle instruction.
4. Confirm first successful launch and baseline monitoring run.
5. Assign first feedback window (24-48h).

## Cadence

- Day 0: cohort kickoff + install verification.
- Day 1-4: active feedback collection, daily triage.
- Day 5: metrics consolidation and go/no-go recommendation.

## Exit criteria for recruitment

- At least 10 active testers with minimum hardware mix reached.
- No single hardware vendor dominates more than 60% of reports.
